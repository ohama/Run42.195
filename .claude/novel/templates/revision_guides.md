# 수정 가이드 템플릿

*품질 검사에서 발견된 문제 유형별 수정 방법*

---

## 사용법

`/novel:check` 실행 후 품질 게이트가 문제를 발견하면, 해당 문제 유형의 수정 가이드를 참조하세요.

---

## 1. 캐릭터 목소리 불일치

### 증상
- 캐릭터가 평소와 다른 말투 사용
- 어휘 수준이 갑자기 달라짐
- 말버릇이 사라지거나 새로 생김

### 수정 단계

```markdown
1. canon/character_voices.md 열기
2. 해당 캐릭터의 음성 프로필 확인
3. 문제 대화문 찾기
4. 다음 체크리스트로 수정:

□ 어휘 수준이 캐릭터에 맞는가?
□ 문장 길이가 평소와 비슷한가?
□ 말버릇이 자연스럽게 포함되어 있는가?
□ 감정 상태에 맞는 말투 변화인가?
□ 성장 단계에 맞는 음성인가?
```

### 수정 예시

**Before (문제):**
```
"나는 이 상황에 대해 심각하게 우려하고 있어."
```

**After (수정):**
```
"이거... 좀 걱정되는데. 진짜로."
```

## 2. 설정 충돌

### 증상
- 캐릭터 외모 묘사 불일치
- 장소/시간 정보 모순
- 이전에 밝힌 정보와 다름

### 수정 단계

```markdown
1. 어느 버전이 "정답"인지 결정
2. canon/ 파일이 정답이면 draft 수정
3. 새 버전이 더 좋으면 canon 먼저 수정

□ canon/characters.md 외모 정보 확인
□ canon/world.md 설정 확인
□ canon/timeline.md 시간 정보 확인
□ 이전 씬에서 같은 정보 어떻게 썼는지 확인
□ 수정 후 관련 씬 모두 검색하여 일관성 확인
```

### 수정 예시

**문제:** ch03에서 "갈색 눈"이 ch07에서 "파란 눈"으로 바뀜

**수정:**
```markdown
1. canon/characters.md 확인: "갈색 눈"
2. ch07 해당 부분 찾기
3. "파란 눈" → "갈색 눈" 수정
4. 다른 씬에서 눈 색 언급 검색: grep "눈" draft/scenes/
```

## 3. 시간선 오류

### 증상
- 날짜/요일 불일치
- 사건 순서 모순
- 불가능한 시간 경과

### 수정 단계

```markdown
1. canon/timeline.md에서 정확한 날짜 확인
2. state/timeline_state.json 참조
3. 실제 달력으로 요일 검증 (일기체의 경우)

□ 이 씬의 날짜가 정확한가?
□ 이전 씬과의 시간 간격이 맞는가?
□ 요일이 날짜와 일치하는가?
□ 계절 묘사가 날짜와 맞는가?
```

### 수정 예시

**문제:** 3월 15일이 화요일이라고 했는데, 실제로는 금요일

**수정:**
```markdown
1. 달력 확인: 2024년 3월 15일 = 금요일
2. 요일 표기 수정: "화요일" → "금요일"
3. 요일에 따른 내용 확인 (학교 수업 등)
```

## 4. 주인공 수동성

### 증상
- 주인공이 사건에 반응만 함
- 결정을 내리지 않음
- 다른 캐릭터가 모든 것을 주도

### 수정 단계

```markdown
1. 씬에서 주인공의 모든 행동 나열
2. 각 행동이 "선택"인지 "반응"인지 분류
3. 최소 1개의 의미 있는 선택 추가

□ 주인공이 씬에서 결정을 내리는가?
□ 그 결정이 스토리에 영향을 미치는가?
□ 주인공이 무언가를 거부/저항하는가?
□ 주인공이 대화를 시작하는가?
```

### 수정 예시

**Before (수동적):**
```
친구가 같이 달리자고 했다. 나는 따라갔다.
비가 오기 시작했다. 우리는 처마 밑으로 피했다.
```

**After (능동적):**
```
친구가 같이 달리자고 했다.
"아니, 오늘은 혼자 뛰고 싶어."
나는 평소 코스 대신 해안 도로를 선택했다.
비가 오기 시작했지만, 나는 계속 달렸다.
빗속에서 뛰는 건 처음인데... 나쁘지 않았다.
```

## 5. 감정 전개 급변

### 증상
- 감정 상태가 갑자기 바뀜
- 충분한 빌드업 없이 클라이맥스
- 감정적 결론이 갑작스러움

### 수정 단계

```markdown
1. 감정 변화의 시작점과 끝점 확인
2. 중간 단계 빌드업이 있는지 확인
3. 부족하면 이전 씬에 힌트 추가 또는 현재 씬에 과정 추가

□ 이 감정에 도달하기까지 몇 단계가 있는가?
□ 각 단계가 독자에게 보이는가?
□ 신체적 반응으로 감정이 표현되는가?
□ 이전 씬들이 이 감정을 예고하는가?
```

### 수정 예시

**Before (급변):**
```
오늘 드디어 5km를 완주했다.
"해냈어!" 나는 기쁨에 겨워 눈물을 흘렸다.
```

**After (점진적):**
```
4km 지점. 다리가 무거워지기 시작했다.
'포기할까...' 잠깐 그런 생각이 스쳤다.
하지만 발을 멈추지 않았다. 한 발, 한 발.

4.5km. 숨이 턱까지 찼다.
저 앞에 5km 표지판이 보였다.

마지막 500m. 세상이 천천히 움직이는 것 같았다.
표지판 앞에 섰을 때, 다리가 풀렸다.
"해냈어..."
눈물이 난 건 기쁨 때문만은 아니었다.
여기까지 오는 데 얼마나 걸렸는지.
```

## 6. 대화 목적 불명확

### 증상
- 대화가 플롯/캐릭터/테마 중 아무것도 발전시키지 않음
- 일상 대화만 있고 갈등 없음
- 정보 전달만 하는 대화

### 수정 단계

```markdown
모든 대화는 최소 2가지를 해야 함:
- 플롯 전진
- 캐릭터 드러내기
- 테마 탐구
- 긴장 생성
- 관계 변화

□ 이 대화가 없어도 이야기가 진행되는가? → 삭제 고려
□ 이 대화에 갈등/긴장이 있는가?
□ 각 캐릭터가 원하는 게 다른가?
□ 대화 후 상황이 변했는가?
```

### 수정 예시

**Before (목적 없음):**
```
"오늘 날씨 좋다."
"응, 그러네."
"뭐 해?"
"별로."
```

**After (목적 있음):**
```
"오늘 날씨 좋다. 뛰기 딱 좋은 날씨야."
"너 요즘 매일 뛰더라. 왜 갑자기?"
잠깐 말이 막혔다. 왜냐고? 나도 잘 모르겠는데.
"그냥... 뛰면 머리가 비워지는 느낌?"
친구가 이상하다는 듯 쳐다봤다.
```

## 7. 감각 묘사 부족

### 증상
- 대화만 있고 물리적 환경 없음
- 시각 외 감각 없음
- "떠다니는 머리" 대화

### 수정 단계

```markdown
씬당 최소:
- 2가지 이상 감각 (시각 + 청각/촉각/후각/미각)
- 물리적 환경 설정
- 캐릭터 신체성 표현 (동작, 자세, 표정)

□ 이 씬의 장소를 독자가 그릴 수 있는가?
□ 시각 외에 다른 감각이 있는가?
□ 대화 중 캐릭터가 무엇을 하고 있는가?
□ 날씨/시간/계절이 느껴지는가?
```

### 수정 예시

**Before (감각 없음):**
```
나는 친구와 이야기했다.
"내일 같이 뛸래?"
"응, 좋아."
```

**After (감각 풍부):**
```
운동장 벤치에 앉았다. 철제 벤치가 햇볕에 데워져 있었다.
바람이 불어와 풀 냄새를 실어왔다.

"내일 같이 뛸래?"
친구가 물병 뚜껑을 돌리며 물었다. 뚜껑 돌아가는 소리가 났다.

"응, 좋아."
나는 운동화 끈을 고쳐 묶었다. 이 신발도 이제 좀 닳았네.
```

## 8. 테마 연결 부재

### 증상
- 씬이 중심 테마와 무관
- 테마적 의미 없는 에피소드
- 단순 일상 나열

### 수정 단계

```markdown
1. canon/premise.md에서 중심 테마 확인
2. 이 씬이 테마를 어떻게 탐구할 수 있는지 고민
3. 직접적 언급보다 상황으로 보여주기

□ 이 씬이 테마와 관련 있는가?
□ 관련 없다면 테마적 층위를 추가할 수 있는가?
□ 캐릭터의 행동/선택이 테마를 반영하는가?
□ 다른 캐릭터와의 대비로 테마를 보여줄 수 있는가?
```

### 수정 예시

**테마:** "작은 것의 누적이 큰 변화를 만든다"

**Before (테마 무관):**
```
오늘 수학 시험이 있었다. 좀 어려웠다.
점심은 급식 카레를 먹었다. 맛있었다.
```

**After (테마 연결):**
```
오늘 수학 시험이 있었다. 좀 어려웠다.
근데 신기한 건, 작년엔 완전 못 풀었을 문제가
이번엔 풀렸다는 거다.

매일 10분씩 복습한 게 효과가 있나?

점심 먹으면서 생각했다.
달리기도 그런 거 아닐까.
매일 조금씩. 어느 날 보니 멀리 와 있는.
```

## 9. 복선 없는 반전

### 증상
- 갑작스러운 정보 공개
- 독자가 "어디서 나온 거야?" 반응
- 뒤늦은 설명 추가

### 수정 단계

```markdown
1. 반전/공개 씬 확인
2. 이전 씬들에서 힌트가 있었는지 확인
3. 없으면 2-3개 힌트 이전 씬에 추가

좋은 복선:
- 처음 읽을 때는 지나침
- 다시 읽으면 "아!" 하게 됨
- 캐릭터 행동/대화에 자연스럽게 녹아있음
```

### 수정 예시

**반전:** ch10에서 할머니가 과거 마라톤 선수였음이 밝혀짐

**추가할 복선들:**

ch03 (힌트 1):
```
할머니 방에 오래된 트로피가 있었다.
"이거 뭐야?" 물었지만 할머니는 그냥 웃기만 했다.
```

ch06 (힌트 2):
```
할머니가 내 달리기 자세를 보더니 뭔가 말하려다 멈췄다.
"아니다, 알아서 배워야지."
```

ch08 (힌트 3):
```
할머니 옷장 정리하다가 오래된 운동복을 발견했다.
번호가 적힌 조끼... 대회 참가 번호 같은데?
```

## 10. 장면 목적 불분명

### 증상
- "이 씬 왜 있지?" 느낌
- 삭제해도 이야기에 영향 없음
- 다른 씬과 기능 중복

### 수정 단계

```markdown
모든 씬은 최소 2가지 기능:
□ 플롯 전진
□ 캐릭터 발전
□ 테마 탐구
□ 긴장 생성
□ 정보 제공
□ 복선 설치
□ 복선 회수

1개만 하면 → 다른 씬과 합치거나 기능 추가
0개면 → 삭제 고려
```

### 수정 방법

**옵션 A: 기능 추가**
```
씬에 갈등/긴장 추가
테마적 층위 추가
캐릭터 관계 발전 추가
```

**옵션 B: 다른 씬과 합치기**
```
두 씬의 핵심 요소만 남기고 하나로 통합
```

**옵션 C: 삭제**
```
정말 필요 없으면 과감히 삭제
대신 필요한 정보는 다른 씬에 분산
```
